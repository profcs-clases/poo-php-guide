01

¿Qué es la POO y por qué nos importa?

Imagina que estás programando de forma "tradicional" (Programación Estructurada). Escribir código es como seguir una receta de cocina paso a paso: cortar, hervir, freír. Funciona bien para cosas pequeñas.

Pero, ¿qué pasa si tienes que construir un videojuego o un sistema bancario? Una receta gigante se vuelve un caos.

La solución: POO

La Programación Orientada a Objetos propone un cambio de mentalidad: en lugar de escribir una receta gigante, vas a crear "cosas" (objetos) que interactúan entre sí, tal como en la vida real.

La analogía del Molde y la Galleta

🔧

Clase

Es el molde para galletas. Define la forma, pero no es la galleta en sí. No puedes comértela.

🍪

Objeto

Es la galleta real creada con el molde. Puedes hacer 100 galletas (objetos) con un solo molde (clase).

02

Tu primera Clase y tu primer Objeto

En PHP, una clase se define con la palabra clave class. El objeto se crea con new.

PHP 
<?php
// Esto es el MOLDE (La Clase)
class Estudiante {
    // Aquí irán las características (Propiedades)
    // Aquí irán las acciones (Métodos)
}

// Esto es la GALLETa (El Objeto)
$estudiante1 = new Estudiante();
$estudiante2 = new Estudiante();
?>

Nota: new es la palabra mágica en PHP que dice "Oye, crea un objeto basado en esta clase".

03

Propiedades y Métodos (Sustantivos y Verbos)

Propiedades (Atributos)

Son las características del objeto (Sustantivos: nombre, edad, carrera).

Métodos

Son las cosas que el objeto puede hacer (Verbos: estudiar, dormir, comer).

PHP 
<?php
class Estudiante {
    // Propiedades
    public $nombre;
    public $edad;
    public $carrera;

    // Métodos
    public function estudiar() {
        echo "¡Estoy estudiando Ingeniería en Sistemas!";
    }
    
    public function presentarse() {
        // La palabra $this es CRUCIAL. Significa "ESTE objeto en particular".
        echo "Hola, soy " . $this->nombre . " y tengo " . $this->edad . " años.";
    }
}

// Uso:
$juan = new Estudiante();
$juan->nombre = "Juan Pérez"; // Asignamos valores
$juan->edad = 19;

$juan->presentarse(); // Salida: Hola, soy Juan Pérez y tengo 19 años.
$juan->estudiar();    // Salida: ¡Estoy estudiando Ingeniería en Sistemas!
?>
04

El Constructor: Dando vida con estilo

Escribir $juan->nombre = "Juan" línea por línea es tedioso. El Constructor es un método especial que se ejecuta automáticamente en el exacto momento en que creas el objeto (new).

PHP 
<?php
class Estudiante {
    public $nombre;
    public $edad;

    // El constructor siempre se llama __construct
    public function __construct($nombre_recibido, $edad_recibida) {
        $this->nombre = $nombre_recibido;
        $this->edad = $edad_recibida;
    }

    public function presentarse() {
        echo "Soy " . $this->nombre . " y tengo " . $this->edad . " años.\n";
    }
}

// Ahora es mucho más limpio y rápido:
$maria = new Estudiante("María Gómez", 20);
$maria->presentarse(); // Salida: Soy María Gómez y tengo 20 años.
?>
05

Encapsulamiento: Protegiendo tus datos 🛡️

Imagina la cápsula de una nave espacial. Los astronautas presionan botones (métodos públicos), pero no tienen acceso directo a los cables y reactores (propiedades privadas).

Modificadores de Acceso:

public

Todos pueden verlo y usarlo desde afuera.

private

Solo la misma clase puede verlo.

protected

Similar a private, pero las clases "hijas" también pueden verlo.

PHP 
<?php
class CuentaBancaria {
    // El saldo es PRIVADO. Nadie puede modificarlo directamente.
    private $saldo; 

    public function __construct($saldo_inicial) {
        $this->saldo = $saldo_inicial;
    }

    // Métodos GETTERS y SETTERS (Son los "botones" de la cápsula)
    public function getSaldo() {
        return $this->saldo;
    }

    public function depositar($monto) {
        if ($monto > 0) { // ¡Aquí protegemos la lógica!
            $this->saldo += $monto;
            echo "Depósito exitoso.\n";
        } else {
            echo "Error: No puedes depositar cantidades negativas.\n";
        }
    }
}

$cuenta = new CuentaBancaria(100);

// $cuenta->saldo = -5000; // ❌ ERROR FATAL. La propiedad es privada.
// $cuenta->saldo += 100;  // ❌ ERROR FATAL.

$cuenta->depositar(50);     // ✅ Correcto. Pasa por la validación.
echo "Saldo actual: $" . $cuenta->getSaldo(); // ✅ Salida: $150
?>
06

Herencia: No te repitas (DRY) ♻️

La Herencia permite crear una clase nueva basándose en una clase ya existente.

Clase Padre (Superclase)

La clase base

Clase Hija 1

Hereda todo

Clase Hija 2

Hereda todo

PHP 
<?php
// CLASE PADRE
class Vehiculo {
    public $marca;
    public $modelo;

    public function __construct($marca, $modelo) {
        $this->marca = $marca;
        $this->modelo = $modelo;
    }

    public function encender() {
        echo "El vehículo está encendido.\n";
    }
}

// CLASE HIJA (Usamos 'extends')
class Auto extends Vehiculo {
    public $cantidad_puertas;

    public function __construct($marca, $modelo, $puertas) {
        // Llamamos al constructor del PADRE para no repetir código
        parent::__construct($marca, $modelo); 
        $this->cantidad_puertas = $puertas;
    }

    // Método exclusivo del Auto
    public function abrirCajuela() {
        echo "La cajuela del " . $this->marca . " está abierta.\n";
    }
}

$miAuto = new Auto("Toyota", "Corolla", 4);
$miAuto->encender();     // ✅ Heredado del Padre
$miAuto->abrirCajuela(); // ✅ Propio del Hijo
?>
07

Polimorfismo: Muchos comportamientos 🎭

"Polimorfismo" significa "Muchas formas". Ocurre cuando clases diferentes tienen un método con el mismo nombre, pero cada una lo ejecuta a su manera.

PHP 
<?php
class Ave {
    public function volar() {
        echo "El ave está volando alto en el cielo.\n";
    }
}

class Pinguino extends Ave {
    // Sobrescribimos (Override) el método del padre
    public function volar() {
        echo "¡Error! Los pingüinos no vuelan, pero nadan muy rápido.\n";
    }
}

class Avion { // No hereda de Ave, pero tiene el mismo método
    public function volar() {
        echo "El avión está volando a 900 km/h.\n";
    }
}

// El polimorfismo brilla cuando los tratas a todos como "cosas que vuelan"
$paloma = new Ave();
$pinguino = new Pinguino();
$boeing = new Avion();

$paloma->volar();   // El ave está volando...
$pinguino->volar(); // ¡Error! Los pingüinos...
$boeing->volar();   // El avión está volando...
?>
08

Clases Abstractas: El Contrato 📜

A veces, una clase es tan general que no debería existir por sí sola. Por ejemplo, la clase FiguraGeometrica. No puedes dibujar una "Figura", puedes dibujar un "Círculo" o un "Cuadrado".

PHP 
<?php
// No se puede instanciar (new Figura() dará error)
abstract class Figura {
    protected $color;

    public function __construct($color) {
        $this->color = $color;
    }

    // Método abstracto: No tiene cuerpo (no tiene código entre llaves).
    // Obliga a las hijas a crearlo.
    abstract public function calcularArea(); 
}

class Circulo extends Figura {
    private $radio;

    public function __construct($color, $radio) {
        parent::__construct($color);
        $this->radio = $radio;
    }

    // Obligados a implementar calcularArea
    public function calcularArea() {
        return pi() * ($this->radio ** 2);
    }
}

$circulo = new Circulo("Rojo", 5);
echo "El área es: " . $circulo->calcularArea();
?>
09

Clase Abstracta vs Interfaz: El duelo definitivo ⚔️

Esta es una de las dudas más comunes en POO. Ambas parecen similares, pero resuelven problemas muy diferentes. Vamos a verlo con una analogía.

La analogía de la Universidad

🏗️

Clase Abstracta

Es como un plano de edificio con algunas paredes ya construidas. El arquitecto dejó partes hechas, pero obligó al constructor a terminar el resto.

"Te doy el 50% hecho, tú terminas lo demás."

📜

Interfaz

Es como un contrato legal. No te da nada construido, solo te dice: "Si quieres llamarte X, obligatoriamente debes poder hacer esto, esto y esto".

"No te doy nada, pero si firmas, debes cumplir TODO."

Tabla comparativa

Característica Clase Abstracta Interfaz
¿Qué es? Una clase incompleta Un contrato de comportamiento
Métodos con código ✅ Sí ❌ No
Propiedades/Atributos ✅ Sí ❌ No
Constructores ✅ Sí ❌ No
Modificadores de acceso public, private, protected Solo public
Herencia Solo de UNA (extends) Puede ser de MUCHAS (implements)
Relación que expresa "ES UN" (identidad) "PUEDE HACER" (habilidad)

Clase Abstracta en acción

Observa cómo tiene propiedades, constructor, un método con código (saludar()) y uno abstracto que obliga a las hijas a implementarlo.

PHP 
<?php
// 🔷 Clase Abstracta
abstract class Empleado {
    // ✅ Puede tener propiedades
    protected $nombre;
    protected $salarioBase;

    // ✅ Puede tener constructor
    public function __construct($nombre, $salarioBase) {
        $this->nombre = $nombre;
        $this->salarioBase = $salarioBase;
    }

    // ✅ Método CON código (implementado)
    public function saludar() {
        return "Hola, soy " . $this->nombre;
    }

    // ✅ Método abstracto (sin código)
    abstract public function calcularSalario();
}

// ❌ ERROR: No se puede instanciar
// $emp = new Empleado("Juan", 1000); 

class Desarrollador extends Empleado {
    private $bono;

    public function __construct($nombre, $salarioBase, $bono) {
        parent::__construct($nombre, $salarioBase);
        $this->bono = $bono;
    }

    // Obligado a implementar el método abstracto
    public function calcularSalario() {
        return $this->salarioBase + $this->bono;
    }
}

$dev = new Desarrollador("Ana", 3000, 500);
echo $dev->saludar();           // ✅ "Hola, soy Ana" (heredado)
echo $dev->calcularSalario();   // ✅ 3500
?>

Interfaz en acción

Observa cómo una clase puede implementar múltiples interfaces (algo imposible con clases abstractas).

PHP 
<?php
// 🔶 Interfaz: solo firma de métodos
interface Volador {
    public function volar();
    public function aterrizar();
}

interface Nadador {
    public function nadar();
}

// ✅ Una clase puede implementar MUCHAS interfaces
class Pato implements Volador, Nadador {
    private $nombre;

    public function __construct($nombre) {
        $this->nombre = $nombre;
    }

    // Obligado a implementar AMBOS métodos
    public function volar() {
        return $this->nombre . " está volando 🦆✈️";
    }

    public function aterrizar() {
        return $this->nombre . " aterrizó en el lago";
    }

    public function nadar() {
        return $this->nombre . " está nadando 🦆🏊";
    }
}

$pato = new Pato("Donald");
echo $pato->volar();      // "Donald está volando 🦆✈️"
echo $pato->nadar();      // "Donald está nadando 🦆🏊"
?>

El ejemplo definitivo: Juntas son más poderosas

En frameworks profesionales como Laravel, verás que se usan ambas en conjunto. La interfaz define el "qué" y la abstracta define el "cómo parcial".

PHP 
<?php
// 🔶 Interfaz: define el "qué" (contrato)
interface Autenticable {
    public function login($usuario, $password);
    public function logout();
}

interface EnviableCorreo {
    public function enviarCorreo($destino, $asunto);
}

// 🔷 Clase abstracta: define el "cómo parcial"
abstract class Usuario implements Autenticable {
    protected $nombre;
    protected $email;

    public function __construct($nombre, $email) {
        $this->nombre = $nombre;
        $this->email = $email;
    }

    // Código compartido (implementado)
    public function logout() {
        return $this->nombre . " cerró sesión 👋";
    }

    // Obliga a las hijas a implementar login
    abstract public function login($usuario, $password);
}

// Clase hija concreta
class Cliente extends Usuario implements EnviableCorreo {
    
    public function login($usuario, $password) {
        return "Cliente $usuario autenticado ✅";
    }

    public function enviarCorreo($destino, $asunto) {
        return "Enviando correo a $destino 📧";
    }
}

$cliente = new Cliente("María", "maria@mail.com");
echo $cliente->login("maria", "1234");
echo $cliente->logout();
echo $cliente->enviarCorreo("admin@mail.com", "Hola");
?>

Regla de Oro (¡memoriza esto!)

¿Las clases comparten código o identidad común?
Usa Clase Abstracta
¿Solo necesito garantizar un comportamiento?
Usa Interfaz
¿Necesito multi-herencia?
Usa Interfaz
¿Necesito propiedades o constructores?
Usa Clase Abstracta

💡 Truco mental:

  • Abstracta = "IS-A" (ES UN tipo de...) → Un Perro ES UN Animal
  • Interfaz = "CAN-DO" (PUEDE HACER...) → Un Perro, un Pájaro y un Avión PUEDEN Moverse
💡

3 Reglas de Oro para el estudiante

1

Nombres con sentido

Tus clases deben ser Sustantivos (Usuario, Factura, Carrito). Tus métodos deben ser Verbos (calcularTotal(), enviarCorreo()).

2

Responsabilidad Única

Una clase debe hacer una sola cosa y hacerla bien. La clase Usuario no debería también conectarse a la base de datos y enviar emails.

3

Practica en Terminal

La POO se entiende viendo los objetos "vivir". Crea un archivo index.php, instancia objetos, imprime sus propiedades con var_dump() o echo y juega con ellos.
